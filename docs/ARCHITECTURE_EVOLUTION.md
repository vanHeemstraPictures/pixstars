# ARCHITECTURE_EVOLUTION.md

## Phase 1 — Functional lamp prototype
- Pi as mic + speaker + LED endpoint
- Mac Mini as HiveMind server
- basic pair-and-speak path

## Phase 2 — Stable stage-ready lamp
- state-based LED embodiment
- wake-word flow
- deterministic Ardour cues
- synthetic voice output from Mac

## Phase 3 — Optimized performance system
- thinner Pi role
- lower-latency audio return path
- richer state coordination
- optional Home Assistant hooks
- optional local wake-word front-end only if it truly helps

## Governing principle

Each phase should reduce stage risk, not increase it.
